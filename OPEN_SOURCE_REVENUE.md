## üìä Monetization Models for Open Source

### 1. **Convenience Selling** (Our Model)
- Source code free on GitHub
- Compiled/packaged versions sold on app stores
- **Examples**: Ardour, Mindustry, Le Coeur du Dragon

### 2. **Dual Licensing**
- Open source for free use
- Commercial license for proprietary use
- **Examples**: Qt, MySQL, GitLab

### 3. **Freemium**
- Core features open source and free
- Premium features paid
- **Examples**: Blender (Blender Cloud), WordPress (premium plugins)

### 4. **Support & Services**
- Software free
- Professional support, training, or hosting paid
- **Examples**: Red Hat, MongoDB

### 5. **Donations**
- Software free with donation options
- **Examples**: Godot Engine ($50K+/month), Blender ($1M+/year)

## ‚ùì FAQ

### Q: "Why would people buy on Google Play if they can get the code free?"

**A:** For convenience! Most users:
- Don't have technical skills to compile code
- Don't have time to set up build environments
- Prefer one-click installation
- Want automatic updates
- Value their time more than $3-5
- Want to support the developer

### Q: "Is it legal to sell open source software?"

**A:** **YES!** 100% legal and encouraged. You're selling:
- The compiled binary
- Convenience of installation
- Professional support
- Automatic updates
- Your time and expertise

### Q: "What license should I use?"

**A:** Depends on your goals:
- **AGPL-3.0** (our choice): Forces sharing modifications, even for web services
- **GPL-3.0**: Forces sharing modifications for distributed software
- **MIT/Apache**: More permissive, allows commercial use without sharing

For games you want to monetize while keeping open source, AGPL-3.0 or GPL-3.0 are recommended.

### 1. Be Transparent
In your Google Play description, clearly mention:
```

üåê FREE VERSION

- One-click installation

```

### 2. Reasonable Pricing
For indie open source games:
- **$0.99 - $2.99**: Very accessible, high volume
- **$3.99 - $5.99**: Standard for premium indie
- **$6.99+**: For games with lots of content

Our choice ($3.86 CAD) is in the low range to maximize accessibility.

### 3. Avoid In-App Purchases
For open source, avoid IAPs because:
- Difficult to manage with open code
- Can create mistrust
- Complicates codebase

Prefer:
- One-time purchase
- Free vs premium version (dual licensing)
- Optional donations

### 4. Multiple Distribution Channels
- **Web**: Free, live demo
- **F-Droid**: Free, open source purists
- **Google Play**: Paid, mainstream users
- **itch.io**: Pay-what-you-want, indie community
